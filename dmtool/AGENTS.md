# Project Guidelines

## Design
- Use a calm, eye-friendly color palette
- Primary color: soft blue
- Background: light gray / off-white
- Text: dark gray (not pure black)
- Avoid harsh contrast and saturated colors
- Style: clean, minimal dashboard
- Use Tailwind CSS
- Use subtle visual hierarchy (clear separation between sections/cards)
- Primary actions and links should use blue accent color
- Avoid a flat gray look — use gentle contrast between elements

## Architecture
- Keep the system lightweight
- Notes are the core entity
- Avoid overengineering
- Build MVP features only unless specified

## Development approach
- Implement one feature at a time
- Do not modify unrelated files
- Keep code simple and readable
## Working Style

* Keep scope tight
* Always do one small step at a time
* Do not redesign existing systems
* Prefer extending current code over refactoring
* Assume you apply changes directly to the codebase
* Do not output full files unless explicitly asked
* Do not output partial code unless necessary
* Keep explanations minimal and practical
* Avoid long discussions or planning
